# Дәрігерлердің жұмыс кестесі мен қабылдау жазбасы

## Жоба туралы
Бұл жоба PostgreSQL және pgAdmin ортасында жасалған деректер қоры болып табылады. Жүйе клиникадағы дәрігерлердің жұмыс кестесін, пациенттерді, қабылдау слоттарын және қабылдау жазбаларын сақтауға арналған.

## Жобаның мақсаты
Жобаның мақсаты — клиникадағы қабылдау процесін автоматтандыру, дәрігерлердің жұмыс уақытын жүйелі түрде сақтау, пациенттердің жазылуын жүргізу, деректердің тұтастығын қамтамасыз ету және SQL сұраныстарын тиімді орындау.

## Пәндік аймақ сипаттамасы
Берілген деректер қоры клиника жұмысына арналған. Жүйеде бөлімдер, дәрігерлер, маманданулар, кабинеттер, пациенттер, қабылдау уақыт слоттары және қабылдау жазбалары қарастырылған. Негізгі идея — дәрігердің бос уақытын және нақты жазылған қабылдауларын бөлек сақтау.

## Негізгі кестелер
- departments — бөлімдер
- rooms — кабинеттер
- specializations — маманданулар
- doctors — дәрігерлер
- doctor_specializations — дәрігер мен мамандану байланысы
- patients — пациенттер
- schedule_slots — дәрігердің қабылдау слоттары
- appointments — қабылдауға жазылу

## Кестелер арасындағы байланыстар
- Бір бөлімде бірнеше дәрігер болуы мүмкін (`departments` → `doctors`)
- Бір дәрігерде бірнеше мамандану болуы мүмкін (`doctors` ↔ `specializations`)
- Бір дәрігерде бірнеше қабылдау слоты болады (`doctors` → `schedule_slots`)
- Бір кабинетте бірнеше слот өткізіледі (`rooms` → `schedule_slots`)
- Бір пациент бірнеше қабылдауға жазыла алады (`patients` → `appointments`)
- Бір слотқа тек бір жазылу сәйкес келеді (`schedule_slots` → `appointments`)

## Пайдаланушылар мен рөлдер
Жүйеде келесі рөлдер анықталған:
- clinic_admin_role — толық басқару құқығы
- registrar_role — пациенттер мен қабылдау жазбаларын жүргізу
- doctor_role — қабылдау статусын жаңарту
- analyst_role — тек оқу құқығы

Құрылған пайдаланушылар:
- clinic_admin
- registrar_user
- doctor_user
- analyst_user

## Тестілік SQL сұраныстар
Жүйені тексеру үшін келесі SQL сұраныстар орындалды:
- бірнеше кестені біріктіретін JOIN сұранысы
- GROUP BY және HAVING сұранысы
- COUNT, SUM, AVG агрегаттық функциялары
- күрделі WHERE фильтрациясы
- EXPLAIN ANALYZE арқылы орындалу жоспарын тексеру

## Қорытынды
